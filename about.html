We outgrew EVE-NG.
So we rebuilt the network lab.

netplex. is designed and built in Australia by engineers who spent years living inside EVE-NG, PNETLAB and GNS3 - and decided the experience around brilliant emulation deserved better.

Why we built it

We got an entire generation of engineers certified on EVE-NG, PNETLAB and GNS3 - and we're grateful for them. But every session started the same way: SSH into a server, juggle image files, run fixpermissions, fight a UI that looked like it shipped in 2007, then open yet another external terminal just to type a command.

The emulation underneath was brilliant. The experience around it hadn't moved in a decade. So we started rebuilding that experience - API-first, in the browser, with a canvas you actually want to look at - on top of the same proven open foundations: QEMU/KVM, Open vSwitch and friends.

"Think GNS3 + EVE-NG + Terraform + NetBox - rebuilt as one modern, API-first, event-driven platform."

What makes it different

It's not just the modern UI. Everything the UI does is a documented API call, so labs become code you keep in your own Git. Pro is a single seat with the complete automation toolbox - no cripple-ware. And our headline differentiator, the multi-host VXLAN stitch fabric (on the roadmap, in beta), spans a single topology across several cheap machines and cloud VMs - so big labs no longer need a big server.
